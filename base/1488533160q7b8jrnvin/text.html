"Apache2 в ОС Astra Linux Special Edition работает только с принудительной аутентификацией".


aptitude install apache2 libapache2-mod-auth-pam


в настройки хоста добавить:

<Directory /var/www/>

AuthPAM_Enabled on

AuthType Basic

AuthName "PAM authentication"

require valid-user

....


a2enmod auth_pam (по умолчанию должен быть уже включен)

service apache2 reload


выдать права на чтение информации из БД пользователей и сведений о мандатных метках:

usermod -a -G shadow www-data

setfacl -d -m u:www-data:r /etc/parsec/macdb

setfacl -R -m u:www-data:r /etc/parsec/macdb

setfacl -m u:www-data:rx /etc/parsec/macdb



для пользователя под которым будем заходить, выполняем команду настройки (по-сути просто сбрасываем в 0)

минимальный и максимальный наборы мандатных категорий (мы их и не настраивали):


usermac -c 0:0 имя_пользователя_под_которым_будем_заходить

(usermac -c 0:0 user)


после этого можно авторизоваться на главной странице.


для других каталогов нужно делать аналогичную настройку в конфигах апача (см. заббикс).
